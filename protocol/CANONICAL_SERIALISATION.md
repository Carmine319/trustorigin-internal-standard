# Canonical Serialisation Doctrine

## Principle

All artefacts must be represented in deterministic structured form.

Identical input must produce identical serialised output.

---

## Objectives

- Eliminate ambiguity in representation
- Enable reproducible integrity hashing
- Ensure cross-environment verification compatibility

---

## Scope

Canonical serialisation applies to:

- Governance artefacts
- Versioned documentation
- AI declarations
- Dependency manifests
- Export bundles

---

## Boundary

Serialisation discipline preserves structure.
It does not validate factual accuracy.
